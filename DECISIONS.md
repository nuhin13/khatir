# Decisions

Architecture and process decisions, newest at bottom. One entry per decision.

- 2026-06-02 · Mono-repo single-repo layout chosen (see architecture 02).
- 2026-06-02 · Docs currently live under `documnets/` (pre-existing, misspelled directory name kept as-is for now; not renamed to `docs/` to avoid breaking existing references). Architecture docs are at `documnets/docs/architecture/`.
- 2026-06-02 · [EPIC-00 T-009] Admin app pins ESLint to `^9.39.4` (latest 9.x), not the newer ESLint 10.4.1. Hard incompatibility: `eslint-config-next@16.2.7` bundles `eslint-plugin-react@7.x`, which calls the `context.getFilename()` API removed in ESLint 10, throwing on every lint run. eslint-config-next's peer range is `eslint >=9`; the Next ecosystem has not yet shipped an ESLint-10-compatible plugin set. Revisit when eslint-config-next supports ESLint 10.
- 2026-06-02 · [EPIC-00 T-009] Tailwind v4 consumes the shared `@khatir/design-tokens` preset via `@config "../../tailwind.config.ts"` in `globals.css`; the config imports the preset through the package `exports` map. The Tailwind/oxide engine resolves this correctly (verified: `bg-sage` compiles to `#7ba084`, etc.). Turbopack emits a cosmetic, non-fatal "Module not found" warning during `next build` because it independently parses the `@config`-referenced file in its own app-rooted module graph and cannot follow the symlinked workspace package's subpath export. The build succeeds and tokens compile; accepted as a known Turbopack limitation.
- 2026-06-02 · [EPIC-00] **uv** is the Python package/runtime manager for `apps/api` (not pip/poetry/pipenv). Single fast resolver, `uv.lock` pins exact versions, and `uv run` lets backend commands and tests run on the host without a container. All Makefile backend targets use `uv run`.
- 2026-06-02 · [EPIC-00] **Latest-stable version policy.** Every app pins the latest *stable* release of its toolchain — no beta/RC. Lockfiles (`uv.lock`, `pubspec.lock`, `package-lock.json`) pin the exact resolved versions for reproducible builds. Documented baselines: Python 3.13, Flutter 3.44 (Dart SDK ≥ 3.12), Node ≥ 20 LTS / Next.js 16. Exception: ESLint pinned to 9.x — see the T-009 entry above.
- 2026-06-02 · [EPIC-00] **Mono-repo, single set of verbs.** All three apps + shared packages + infra live in one tree, driven by one `Makefile` and one `docker-compose.yml`. Rejected per-app repos to keep cross-cutting changes (design tokens, conventions, CI) atomic and to let one tracker/board span the whole product.
- 2026-06-02 · [EPIC-00 T-010] **Design tokens are a single shared source** (`packages/design-tokens`) consumed by both the Flutter app (Dart token file) and the admin (Tailwind preset via the package `exports` map). Colors/spacing/radius/type are never hardcoded in app code; they come from this package so a token change propagates to mobile + web at once.
- 2026-06-02 · [EPIC-00 T-006] **Celery runs eagerly in tests** (`task_always_eager` / eager result backend in the test settings). Tasks execute synchronously inline during `pytest`, so the suite needs no live broker/worker and assertions can read task results directly. Real async execution (worker + beat) is used only at runtime via docker compose.
