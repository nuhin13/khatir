02Tokens

Colour system

Three brand hues at a shared low chroma — sage (trust / primary), dusty rose (warmth / CTA & alerts), butter (highlight) — over warm cream and soft ink. Each hue carries a tinted background for calm fills.

Every saturated surface keeps WCAG-AA text contrast. Rose doubles as the alert colour; a separate --danger #D14D3B is reserved for destructive/kill-switch actions in admin.

03Tokens

Typography

Plus Jakarta Sans for Latin titles & numerals, Hind Siliguri for all Bangla body, Caveat for sparing handwritten warmth on hero/empty states. Minimum body size 14px; tap-target labels 13px+ for older users.

04Tokens

05System

Icons — the hybrid rule

Line icons (1.8px stroke, rounded caps) carry all navigation, actions and status — consistent and crisp at any size. Emoji are reserved for hero illustrations, empty states and celebratory moments, where their warmth is an asset, not noise.

08Engineering

Handoff notes

Flutter · mobile
  • Tokens → ThemeData + a KColors / KText constants file.
  • Cards = Card w/ 22px radius, soft shadow; buttons = FilledButton stadium shape.
  • Bangla default locale; English via flutter_localizations.
  • Large 44px+ tap targets; offline draft + sync for all forms.
Next.js · admin
  • Desktop-first, calmer ops palette (same hues, cooler neutrals).
  • MFA-required; RBAC across 12 console sections.
  • Separate subdomain ops.khatir.com.bd.
  • Admin reference UI ships in the next pass.
Hard constraints
  • No public listings or reviews — ever. All reputation is private, factual, consent-gated.
  • NID data BD-hosted, encrypted; store result not raw payload.
  • Kill-switch + right-of-reply on every warning/flag.
  • Pricing fully admin-configurable, no hard-coded prices.
